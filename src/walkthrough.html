Key Actions

  • Refresh: Reloads the dashboard data.

What You See

A comprehensive table of all bookings with columns for Ref #, Client, Date, Package, Status, and Payment.

Key Actions

  • Update Status: Change status to Confirmed, Completed, or Cancelled.

Automated Actions

  • Event Done: Past confirmed events auto-update to Post Production.
  • Photo Upload: Uploading photos auto-updates status to Completed.

User Module

Located in src/user/. This is the client-facing portal.

What You See
  • Summary Cards: Upcoming Events count, Total Bookings, Total Spent.
  • Chart: Bar chart showing your bookings by package type.

Inputs & Validations
  • Event Date: Must be at least 3 days in advance. Cannot select already booked dates.

  • Payment Proof: File upload required for non-cash payments.

Financial Features

System automatically generates invoices when a booking is confirmed. Invoices can be downloaded as PDF.

Handled by api/generate_invoice.php
Refunds

Automated refund requests upon cancellation. Admins process refunds offline and upload proof.

Handled by admin/api/refunds.php
Balance Payments

Users can pay remaining balances via the portal by uploading payment proof. Admins verify and confirm.

Handled by user/api/payment_api.php

Comprehensive Process Guide

Step 1: Booking (User)

User fills out the booking form, selects a package, and pays the downpayment.

Step 2: Confirmation (Admin)

Admin reviews the booking and downpayment proof. Upon approval, status becomes Confirmed.

Step 3: The Event

The photoshoot/event takes place as scheduled.

Step 4: Photo Upload (Admin)

Admin selects the best photos and uploads them via the Photo Upload page, assigning them to the specific booking.

Step 5: Gallery Access (User)

User logs in, goes to My Photos, and can view/download their exclusive gallery.

Downpayment Rules

The system requires a minimum downpayment of 25%. If a user pays more than 25%, the system automatically deducts the full amount paid from the total price. The remaining balance will be lower.

How to Pay Remaining Balance
  1. User logs in and goes to My Appointments.
  2. Find the booking with a pending balance.
  3. Click the Pay Balance button.
  4. Upload the proof of payment (screenshot/receipt).
  5. Status updates to Payment Pending Confirmation.
  6. Admin verifies the proof in Manage Bookings and clicks Confirm Payment.

Refunds are handled securely with proof verification.

Initiation

When a booking is cancelled (by User or Admin), a Refund Request is automatically generated if a payment was made.

Processing (Admin)

Admin goes to Refunds page, reviews the request, and processes the refund offline (e.g., bank transfer).

Proof Submission (Admin)

Admin marks the refund as Processed and uploads the transaction receipt/proof.

Verification (User)

User receives a notification. They can view the status and the uploaded proof in their Billing History page.

The system keeps users informed about important updates.

  • Bell Icon: Located in the top header. It shows a red badge count for unread notifications. Clicking it reveals a dropdown with the latest updates.
  • Sidebar: The sidebar highlights the current page you are on, but does not show notification badges. Users should rely on the header bell icon.
  • Email: Critical updates like Booking Confirmation are also sent via email.

API Structure

The system uses a REST-like API architecture for dynamic data fetching.

Key Endpoints
  • GET admin/api/manage_booking.php - Fetch booking details
  • POST admin/api/manage_booking.php - Update status/notes
  • GET user/getAppointments.php - Fetch user bookings
  • POST user/api/payment_api.php - Upload balance payment proof
  • GET user/api/billing_api.php - Fetch financial history
